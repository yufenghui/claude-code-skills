# 8X Flow 产出图固化设计

> 日期：2026-06-30
> 范围：`packages/business-modeling/skills/bm-8x-flow`
> 目标：把 8X Flow 五步法各环节应有的建模图固化进 skill 的产出章节序列，补齐当前缺失的图输出。

## 一、背景与问题

`bm-8x-flow` skill 的五步法在多个环节要求建模图输出，但现有产出章节序列存在三处缺口（均有 8X Flow 原文依据）：

### 缺口 1：凭证追溯图缺失（步骤 2）

- 步骤 2「找主要履约项，按四色找凭证」的找凭证子步骤③要求"追溯每个关键数据项的来源"。
- 原文依据：第 15 讲第 75 行"在这条时间线上**追溯**履约中的关键凭证"；`concepts.md`「关键数据项的三类来源」（前序凭证数值 / 人工输入 / 规则计算）。
- 现状：产出"主要履约项表"只有**表格**（权利方/义务方 + 请求/确认 + 凭证名），未把追溯结果画成图。
- `bm-four-color` skill 已有现成画法可复用：**凭证表 + 凭证链图**（实线 = 数值/时标来源，虚线 = 规则来源），即"业务脊梁"。

### 缺口 2：最终业务模型全视图缺失（收尾）

- 原文依据：第 16 讲第 191–203 行——把三个合同上下文（专栏订阅 / 作者协议 / 编辑绩效）放一起，凭证跨上下文串联，就是"**一套相对完整的业务模型**"全视图，展示"极客时间通过哪些运营活动将一个 CMS 转化为可盈利的产品"。
- 现状：现有 skill 把它**拆成了**合同上下文图、违约流转图、弹性边界图等分项图，反而**没有一张整合的全景图**。
- `output-format.md` 第 31 行其实已要求"业务模型图（至少一张全景图 + 关键关系图）"，但 8X Flow 的章节序列一直未落实。

### 缺口 3：变化点缺图

- 原文依据：第 15 讲第 169 行，变化点（履约确认角色化）是**有图**的。
- 现状：产出仅"变化点清单"（SKILL 第 102 行），无图。

## 二、决策（已与用户逐项确认）

| 决策项 | 选定方案 | 理由 |
|--------|---------|------|
| 凭证追溯图形态 | **独立章节**，放步骤 2 | 与"主要履约项表"并列，追溯结果单独成图更清晰 |
| 凭证追溯深度 | 只追**主要履约项核心凭证链**；完整全量仍调 `bm-four-color` | 保持两 skill 分工，避免在 8X Flow 内重做整套四色 |
| 最终业务模型全视图位置 | **收尾独立章节**，放弹性边界/服务图之后 | 全视图需等所有合同上下文建模完成才能整合（16 讲"终极形态"）；弹性边界图聚焦服务映射，视角不同，保留 |
| 变化点形态 | **图为主 + 表为辅** | 角色化履约确认图为主，关键变化点表补充 |
| 端到端走查位置 | 全视图**之后** | 先整合出完整模型，再走查验证，作为模型成型后的验证收尾 |
| 违约凭证追溯 | **不单独追溯** | 凭证追溯图只覆盖主要履约项核心链；违约履约项凭证在违约流转图节点上标注，保持轻量 |

## 四、各新增 / 升级章节内容定义

### 第 3 章 · 凭证追溯图（新增）

- **追溯对象**：第 2 章主要履约项的凭证。
- **追溯内容**：每个关键数据项（通常是时间点 + 金额）的来源——
  - ① 前序凭证的数值（如支付金额 ← 订单金额）
  - ② 人工输入（如编辑输入报价）
  - ③ 由前序凭证中的规则/算法计算得到（如骑手配送费 = f(距离, 时段, 天气)，追溯到计费规则凭证）
- **图形式**（复用 `bm-four-color` 凭证链图画法）：
  - 节点 = 凭证（标关键数据项）
  - 实线 `-->` = 数值 / 时标来源
  - 虚线 `..>` = 规则来源
  - 在线 / 节点上标注**计算逻辑**与**时标顺序逻辑**（如 `Order.placed_at > ColumnQuote.available_at`）
- **配套**：凭证表（关键数据项 + 来源类型 + 计算/时标逻辑）
- **边界**：只追主要履约项核心链；违约履约项凭证在第 4 章违约流转图节点上标注，不入此图；如需完整全量凭证链，调用 `bm-four-color`。

### 第 5 章 · 变化点图（升级，原"变化点清单"）

- **图（为主）**：角色化履约确认图——哪些履约确认被角色化、由哪些合同上下文的凭证来扮演（呼应第 15 讲第 169 行，跨合同上下文的凭证引用 = 依赖关系，反转依赖后引入变化点）。
- **表（为辅）**：关键变化点清单——变化点 / 被角色化的履约确认 / 扮演凭证的来源合同 / 业务意义。

### 第 7 章 · 最终业务模型全视图（新增）

- **整合内容**：所有合同上下文 + 凭证跨上下文串联 + 领域系统（标的物归领域系统，通过凭证引用参与业务）。
- **呈现意图**：业务如何通过运营活动将领域系统转化为可盈利的产品（第 16 讲第 203 行）。
- **场景适配**：
  - 单合同场景：全视图即该合同上下文的完整业务模型（合同 + 主要履约项 + 凭证 + 违约 + 变化点 + 领域边界）。
  - 多合同场景：多个合同上下文经凭证串联的整合全景图（如专栏订阅 × 作者协议 × 编辑绩效）。
- 该章节落实 `output-format.md` 第 31 行"业务模型图（至少一张全景图）"在 8X Flow 的要求。
