# 性能报告

生成详细的性能分析报告，包括会话概览、效率指标、慢操作 TOP5、失败分析和模块耗时分布。

## 使用方法

运行演示报告（使用模拟数据）：

```bash
cd packages/ai-insights
node demo-report.mjs
```

**注意：** 由于当前 Claude Code 的 `history.jsonl` 仅记录用户消息，
不包含工具调用详情（工具名、参数、结果、耗时等），因此暂时无法从
实际日志中提取性能数据。

演示报告使用模拟数据展示报告功能，包含：
- 67 个模拟操作（Read、Edit、Bash、Glob、Grep）
- 各种耗时分布（50ms - 45s）
- 8 个失败操作，3 种错误类型

## 报告内容

- **会话信息**: 日期、时长、总操作数
- **效率指标**: 成功率、平均耗时、效率评分
- **慢操作 TOP5**: 最耗时的 5 个操作
- **失败分析**: 失败操作统计和原因聚合
- **模块耗时分布**: 各模块的耗时和操作数

## 示例输出

```markdown
# 📊 性能报告

## 会话信息
- **日期：** 2026-01-20
- **时长：** 45 分钟
- **总操作数：** 127

## 效率指标
- **成功率：** 94.5%
- **平均耗时：** 1.8s
- **效率评分：** 85/100

## 慢操作 TOP5
1. Bash "npm install" - 45.2s 🔥
2. Read node_modules/... - 12.3s 🐌
...
```

## 备用方案

如果 CLI 不可用，可以手动读取缓存数据：

```bash
cat ~/.claude/ai-insights/current-session.json
```
