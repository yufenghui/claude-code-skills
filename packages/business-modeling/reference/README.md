## 课程脉络（一句话）

- **旧约·前云时代（DDD）**：知识消化（两关联一循环）→ 跨越现实的障碍（4 个实现模式）→ 让模型当统一语言（催化剂/角色-目标-实体/事件风暴/四色法）→ 落地为 RESTful API。
- **新约·云时代**：弹性优先原则 → 区分领域/业务逻辑 → **8X Flow（合同履约法）** → 中台（宏流程提取）→ 微服务落地 → SaaS（魔球建模法）。

## 一条贯穿全课的主线

> **业务建模首先是定义问题的方法，其次才是解决问题的方法。** 从业务本身的结构（财务权责、合同履约）出发去建模，而不是从技术方案反推问题。

四色法的"凭证追溯"与 8X Flow 的"合同履约"同源——都来自**财务审计/合同审计**看企业经营的视角：不了解领域，也能凭权责与合同快速、系统地理解业务。这是本课程最核心的洞察。
