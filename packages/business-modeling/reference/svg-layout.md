# SVG 业务图布局指南

> 业务建模报告里手写 SVG 图的布局规范。目标：图清晰、连线不遮挡、逻辑分组不乱。

## 核心心法

**遮挡的根因是“连线穿过节点”，不是 z 序。** 先保证连线不穿越任何节点（走通道），再用 z 序（先线后点）让端点干净。绝不为避开穿越而打乱逻辑分组。

## 五条布局原则

1. **逻辑分区**：节点按职能 / 角色分组，同组同区（同行 / 同列 / 同框），用位置或背景色区分。
2. **流向一致**：主依赖 / 流程方向统一（自上而下 或 自左而右），不来回穿插。
3. **连线走通道，不穿节点**：连线只走节点行 / 列之间的留白通道；用短斜线或横平竖直折线（曼哈顿路由），绝不穿过节点矩形。
4. **z 序：先连线后节点**：先画连线（底层），后画节点（上层）——节点覆盖连线端点，视觉干净。
5. **就近 + 简化**：强耦合节点相邻缩短连线；只画核心关系线，次要关系交给表格 / 文字。

## 绘制顺序（z 序，从底到顶）

1. 背景 / 分区底框
2. 连线（`<path>`）
3. 节点（`<rect>` + 色帽 + `<text>`）
4. 连线标注 / 图例

> 节点在上层覆盖连线端点；连线标注放最上层，避免被节点压住。

## 坐标规划（网格 + 通道法）

- `viewBox` 推荐 `0 0 1040 H`（宽固定 1040，高按行数）。
- **行布局**：每行节点等高，**行间留通道 ≥ 40px** 供连线横走。
- **列布局**：节点等宽，列间留通道供连线竖走。
- 节点中心对齐网格，便于连线对齐。

行布局速查（节点高 80，通道 54）：

| 行 | 节点顶 y | 节点底 y | 通道 |
| --- | --- | --- | --- |
| 1 | 62 | 142 | 142–196 |
| 2 | 196 | 276 | 276–326 |
| 3 | 326 | 406 | — |

横向 5 节点速查（宽 180，间距 40，左边距 40）：x = 40, 260, 480, 700, 920；cx = 130, 350, 570, 790, 910。

## 连线路由选择

| 场景 | 路由 | 说明 |
| --- | --- | --- |
| 相邻同列节点 | 垂直直线 | 最短 |
| 相邻同行节点 | 水平直线 | 最短 |
| 相邻斜对角 | 短斜线 | 跨 1 个通道 |
| 跨多节点 | 曼哈顿折线 | 走通道，横平竖直 |
| 绕行 / 汇聚 | 三次贝塞尔 | 控制点取中段 |
| 分组间汇总 | 一条主线 + 文字 | 避免多线穿越 |

**铁律**：每条连线，沿路径检查不经过任何节点矩形内部。

## 按图类型的布局模式

- **流向图（服务图 / 流程图）**：分行 / 分列，单向流向；外部依赖放相邻行（如 业务 → 外部 → 领域 三行）。
- **网格图（CBM 热力图）**：行列严格对齐，表头行 / 列，单元格等大。
- **时序图**：参与方纵向生命线，横向消息，`Note` 标在列间。
- **关系图（合同 / 组件总图）**：核心节点居中、周边辐射；或分层（业务层 → 领域层）。
- **凭证链**：时标型横向主链（**专用坐标以 [`html-report.md`](./html-report.md) §五 为准**，本页横向速查仅为通用示例），描述型规则在上一行、虚线下指。

## 常见错误 → 对策

| ❌ 错误 | ✅ 对策 |
| --- | --- |
| 连线穿过节点（被遮挡） | 走通道 / 曼哈顿折线绕行 |
| 为避让打乱逻辑分组 | 保留分组，连线走行间通道 |
| 先画节点后画连线（连线压住节点文字） | 先连线后节点 |
| 流向混乱、连线交叉多 | 统一流向，强耦合就近 |
| 连线起止悬空或深入节点内部 | 起止于节点边缘（留 2–3px） |
| 通道太窄、多条线挤一起 | 行间 / 列间通道 ≥ 40px，多条线错开 y |
| 一图画所有关系（蜘蛛网） | 只画核心关系，次要靠表格 / 文字 |

## 画图前自检清单

- [ ] 确定图类型 → 选布局模式
- [ ] 规划分区与流向
- [ ] 网格坐标 + 行间 / 列间通道 ≥ 40px
- [ ] 先画连线，后画节点
- [ ] 每条连线检查：不穿任何节点
- [ ] 连线起止于节点边缘
- [ ] 强耦合节点就近
- [ ] 只画核心关系线
- [ ] 图例 + 标注在最上层
